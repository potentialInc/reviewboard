# Prompts to AURA

Use Playwright to execute design prompts on AURA.build and generate HTML screens.

## Prerequisites

1. **Playwright MCP** or **playwright-cli** installed
2. **AURA.build account** — keep logged in
3. **Design prompts file** — from `prd-to-design-prompts` or `design-guide-to-aura-prompts`

## Usage

```
prompts-to-aura <prompts-file>
prompts-to-aura <prompts-file> --output ./my-screens
prompts-to-aura <prompts-file> --project "Custom Name"
```

| Argument | Description | Default |
|----------|-------------|---------|
| `<prompts-file>` | Path to AURA prompts markdown | (required) |
| `--output` | HTML output directory | `./generated-screens/{project}/` |
| `--project` | Override project name | from prompts file |

## Prompts File Format

Standard format generated by the design prompt skills:

```markdown
---
project_name: "Project Name"
total_pages: 11
design_system:
  style: "dark"
  primary_bg: "zinc-950"
  font: "Inter"
---

## Design System
[Common prefix for all pages]

---

## Page: 01-landing
name: Landing Page
category: marketing
[Page-specific prompt]

---
```

### Parsing Rules

| Element | Method | Usage |
|---------|--------|-------|
| `project_name` | YAML frontmatter | AURA project name |
| `## Design System` | Section content | Common prompt prefix |
| `## Page: [slug]` | Regex match | HTML filename |
| `name:` | First line after header | AURA page display name |
| `---` | Delimiter | Page separator |

## Workflow

### Step 1: Parse Prompts File

1. Read YAML frontmatter for project metadata
2. Extract `## Design System` section as common prefix
3. Split remaining content by `## Page:` sections
4. For each page: extract slug, name, category, and prompt body

### Step 2: Open AURA.build

```bash
# Using playwright-cli (preferred)
playwright-cli -s=aura-gen open https://aura.build --persistent

# Or Playwright MCP
mcp__playwright__browser_navigate → https://aura.build
```

### Step 3: For Each Page

1. Navigate to AURA's prompt input
2. Enter: `[Design System prefix]\n\n[Page prompt]`
3. Wait for generation to complete
4. Download the generated HTML
5. Save to `{output-dir}/{page-slug}.html`

### Step 4: Verify Output

```bash
ls -la {output-dir}/*.html
# Should match total_pages count from prompts file
```

### Step 5: Report Results

```
Generated HTML screens:
- 01-landing.html (12KB)
- 02-login.html (8KB)
- 03-signup.html (9KB)
...

Output: ./generated-screens/{project}/
Total: {N} pages generated

Next steps:
- aura-to-git (deploy to GitHub Pages)
- set-html-routing (fix navigation)
```

## Rules

- Prefix every page prompt with the Design System section
- Save HTML with the exact slug from `## Page: {slug}`
- Create output directory if it doesn't exist
- Never overwrite existing files without confirmation
- Use playwright-cli sessions for token efficiency

## Error Handling

- Prompts file not found: **STOP** and suggest running design prompt skills first
- AURA.build not accessible: check internet, report error
- AURA generation fails for a page: skip, note in report, continue with next
- Playwright not available: **STOP** and report setup instructions
- Partial generation: report which pages succeeded/failed
